# Style-to-Theme Mapping

Use this file when a `drawio-skill`-style preset is active and `drawio-lei` must map it onto the `drawio-bahayonghang` theme system.

## Precedence

Resolve visual defaults in this order:

1. explicit theme from the user
2. mapped theme from the active preset
3. engineering default theme `tech-blue`

After selecting the base theme:
- use the theme for canvas, module containers, token defaults, and any semantic types the preset does not override
- use the preset for palette intent, role mapping, shape vocabulary, font choices, edge defaults, and extras such as `sketch` and `globalStrokeWidth`

## Built-in Mapping

| Preset | Base Theme | Why |
|------|-------------|-----|
| `default` | `tech-blue` | Closest engineering default: blue-led palette, rounded service boxes, clean architecture look |
| `corporate` | `high-contrast` | More rigid enterprise framing: squared shapes, Arial-like feel, stronger borders |
| `handdrawn` | `nature` | Softer and less formal baseline that works with sketchy overlays |

## Overlay Rules

### Nodes

- Keep the semantic node type from the YAML spec.
- Preserve structural shape rules from diagram-type presets such as ERD tables, UML lifelines, or swimlanes.
- When the preset defines a compatible shape keyword, use it as a style override.
- When the preset defines a role color, convert it into explicit `style.fillColor` and `style.strokeColor` on the node.
- When the preset defines fonts, set `style.fontFamily`, `style.fontSize`, and title/header overrides where relevant.

### Edges

- Start from the mapped theme connector type.
- Overlay `preset.edges.style`, `preset.edges.arrow`, dashed behavior, and global stroke width.
- Keep routing coordinates and waypoints generated by the engineering workflow.

### Modules and Canvas

- Keep module containers and canvas background on the mapped theme by default.
- If the user explicitly wants preset colors on containers too, map:
  - service-heavy module -> `primary`
  - data module -> `success`
  - queue/integration module -> `warning`
  - gateway/edge module -> `accent`
  - external/boundary module -> `neutral`

## Missing Values

If the preset is incomplete:
- inherit missing node/module/canvas values from the mapped theme
- do not invent new preset slots
- warn only when the missing field materially changes the output

## Theme Selection Shortcuts

When the user does **not** name a preset but asks for a broad mood, choose:
- `modern`, `engineering`, `architecture` -> `tech-blue`
- `accessible`, `strong contrast`, `printer safe` -> `high-contrast`
- `organic`, `soft`, `sketchy` -> `nature`
- `dark`, `presentation`, `terminal style` -> `dark`

Do not auto-select `academic` or `academic-color` in this skill. This skill stays engineering-first unless the user explicitly names those themes.
